Treatball Roller Game

Treatball Roller (placeholder name) is a simple casual game built in Unreal Engine 5 where you play as a dog. The objective is to roll a ball around that holds treats, and collect treats that fall out. There will be many levels, dogs to choose from, treatballs to choose from, and more! This game is currently in development, updates will be posted on this page.

Oh boy, here I go updating again!

New stuff:

  • Added a Title Screen that disappears when clicked
  • Added a menu button in the top right corner of the screen that pauses the game
  • Within that menu are various buttons including settings, return to title, respawn ball at start, and an X button
  • The settings menu has a non-functional slider to adjust the music that doesn't exist
  • The return to title returns the player to the title screen
  • The respawn ball at start deletes the ball and respawns it where the level began (in case of it disappearing or getting stuck)
  • The X button, you guessed it, exits the menu and resumes play
  • I managed to create two levels and start on a third

Set backs and bugs:
  • It took me a lot of time trying to figure out the respawn ball feature. I thought this would be a good idea because while play testing the ball clips through things or gets stuck in awkward spots.
  • When I go into the Menu for the second time, the Menu button does not delete (so it can still be seen by the X). This is not as designed and should hide itself while in the Menu.
  • Using the Return to Title button does not have the click to 'Begin Play'
  • I've made a few attempts at importing a human model (owner of the dog), but the material looks metallic and the model isn't smooth.

What's next:
  • Fix the above bugs
  • Level selection in the title screen
  • Finish up third level design
  • Succesfully import the Owner model
  • Add music and attach it to the volume slider
  • Stretch goal: add some AI to the Owner NPC

I believe that is everything. I'm happy with how much I got done! See you next time!

First update! Development started this month on my first game! I'll go over updates to the game, any set backs I'm facing, and what is planned for next month. Now! Onto the fun stuff!

New stuff:

  • Imported dog and ball assets
  • Added movement input to the dog so it can be controlled via WASD
  • Added animations to the dog's movement
  • Added an idle animation to the dog
  • Set up collision physics so the ball moves appropriately when the dog boops it
  • Added a boop animation when the dog runs into the ball nose first
  • Created and imported a cereal treat asset
  • Added FX and floating animation to the cereal treat
  • Added functionality to the treatball that drops a treat ball everytime a point on the ball creates a collision event with the dog
  • Added a point system to the ball and UI for it on the screen

Set backs and bugs:
  • Had some challenges with the nose boop animation, still not perfect on timing, but plan on refining it later
  • I had to replace my original treatball asset due to it not importing from Blender correctly
  • When I first set up the treat to spawn, it spawned several at a time. This was fixed
  • Stuggled a bit with setting up the UI. Getting it placed on the screen right, functioning correctly, etc.

What's next:
  • Title screen with level selection
  • First level design
  • Pause menu with a "Back to Title Page", "Settings", and exit buttons
  • Volume slider in the settings menu (sound and fucntionality to come later)

That's all I have for now! Got a lot done in the first month, and I am excited to share more with you! Thank you!
